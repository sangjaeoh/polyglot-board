# 게시판 요구사항 (Requirements)

Spring Boot 기반 게시판 최소 구현. 익명 사용자가 게시글을 쓰고 읽는 핵심 흐름의 요구사항(*무엇을 만드는가*)을 정의한다. 실제 구현 모델(도메인·엔티티·필드·정책·불변식)은 [`DOMAIN_MODEL.md`](./DOMAIN_MODEL.md)가 소유한다.

- 대상 사용자: 익명 사용자 — 인증 없음.
- 기준 방향: 단일 도메인(board) 모놀리스. API 앱과 마이그레이션 앱을 분리하고, 계약은 OpenAPI 스냅샷이 원천이다.

## 범위

- 포함: 게시글 작성, 게시글 상세 조회, 최신순 페이지 목록 조회, 게시글 수정, 게시글 소프트삭제.
- 제외: 인증·인가, 댓글, 검색, 작성자 소유권 검사(작성자 문자열과 요청 주체를 잇는 수단이 없다 — 보류 판정과 정합).

## 용어

이 표가 도메인 용어집을 겸한다.

| 용어 | 정의 |
|---|---|
| 게시글(Post) | board 도메인의 유일한 애그리거트. 제목·본문·작성자를 갖는 글 1건 |
| 제목(title) | 게시글의 제목. 최대 200자 |
| 본문(content) | 게시글의 내용. 최대 10000자 |
| 작성자(author) | 작성자 표시 문자열. 최대 20자. 계정이 아니라 작성 시점 스냅샷 |
| 소프트삭제 | 물리 DELETE 없이 `deletedAt` 세팅으로 표현하는 논리삭제 |
| 활성 게시글 | 소프트삭제되지 않은(`deletedAt IS NULL`) 게시글. 모든 조회·수정·삭제의 대상 |
| 최신순 | 생성시각 내림차순, 동시각은 id 내림차순 |

## 기능 요구사항

### 게시글

- 제목·본문·작성자로 게시글을 작성할 수 있다. 세 값은 모두 필수다.
- 게시글 상세(제목·본문·작성자·작성/수정 시각)를 조회할 수 있다.
- 활성 게시글을 최신순으로 페이지 조회할 수 있다. 목록 항목은 본문을 제외한 가벼운 형상(제목·작성자·작성 시각)이다.
- 게시글의 제목·본문을 수정할 수 있다. 작성자는 불변이다.
- 게시글을 소프트삭제할 수 있다. 삭제된 게시글은 모든 조회에서 제외되고, 상세 조회·수정·삭제 시도는 미존재와 같게 거부된다.
- 없거나 삭제된 게시글에 대한 상세 조회·수정·삭제는 `POST_NOT_FOUND`(404)로 거부한다.

## 비기능 요구사항

- 페이지네이션: 요청·응답은 1-based(page ≥ 1, size 1–100, 생략 시 page 1·size 20), 도메인 내부는 0-based. 응답 전체 항목 수는 JSON number 정밀도 제한을 피해 문자열로 싣는다.
- 에러 계약: 모든 오류 응답은 RFC 9457 ProblemDetail이며 기계 분기용 `code` 확장 멤버와 관측용 `traceId`·`instance`를 싣는다. 검증 실패는 400 `VALIDATION_FAILED`와 필드 오류 목록으로 응답한다.
- 계약 원천: HTTP 계약은 OpenAPI 스냅샷(`openapi.json`)이 원천이며 drift 게이트가 코드와의 불일치를 빌드에서 막는다.
- 이력·복구: 삭제는 소프트삭제라 데이터가 보존된다(복구 오퍼레이션은 범위 밖).

## 제약·전제

- 작성자는 자유 문자열 스냅샷이다 — 회원·계정 모델이 없고, 같은 문자열이 같은 사람임을 보장하지 않는다.
- 단일 DB·단일 스키마(`board`)를 전제한다. 물리 FK 없음·인덱스는 마이그레이션 소유 등 영속성 규칙은 `docs/entity-persistence.md`를 따른다.
- 마이그레이션은 별도 앱(app-migration)이 수행하고 API 앱은 스키마를 검증만 한다(ddl-auto validate).
